Définition de la virtualisation

La virtualisation consiste, en informatique, à exécuter sur une machine hôte, dans un environnement isolé, des systèmes d'exploitation, on parle alors de virtualisation système ou des applications on parle alors de virtualisation applicative. Ces ordinateurs virtuels sont appelés serveur privé virtuel (Virtual Private Server ou VPS) ou encore environnement virtuel (Virtual Environment ou VE).

Différence entre émulateur, simulateur et outil de virtualisation

Émulateur

Définition :

Un émulateur est un logiciel ou un système qui reproduit fidèlement le fonctionnement d'un matériel ou d'un système dans un environnement différent.

Exemples :

  • Émulateur Android (comme BlueStacks) pour exécuter des applications Android sur un PC.
  • Émulateurs de consoles de jeux (comme PCSX2 pour la PlayStation 2).

Caractéristiques :

  • Imite à la fois le matériel et le logiciel.
  • Moins performant que le matériel réel (car il reproduit chaque instruction à bas niveau).
  • Utilisé pour tester des applications sur différentes plateformes matérielles.

Simulateur

Définition :

Un simulateur est un logiciel qui recrée un environnement ou un comportement spécifique sans imiter le matériel sous-jacent.

Exemples :

  • Simulateurs iOS (fournis par Xcode) pour tester des applications iPhone sur macOS.
  • Simulateurs de vol pour l'entraînement des pilotes.

Caractéristiques :

  • Simule uniquement le comportement logiciel (sans imiter le matériel).
  • Plus rapide qu'un émulateur, mais moins précis.
  • Utilisé principalement pour les tests d'applications.

Outil de virtualisation

Définition :

Un outil de virtualisation crée un environnement virtuel pour exécuter plusieurs systèmes d'exploitation ou applications sur une seule machine physique.

Exemples :

  • VirtualBox, VMware, ou Hyper-V pour exécuter des machines virtuelles (VM).
  • Docker pour virtualiser des applications via des conteneurs.

Caractéristiques :

  • Ne cherche pas à reproduire un système différent, mais à isoler plusieurs systèmes ou applications sur une même machine.
  • Permet l’exécution parallèle de systèmes d’exploitation (ex. : Linux et Windows sur un même PC).
  • Utilisé pour le développement, les tests, et la production dans des environnements partagés.

Type de virtualisation pour les environnements VirtualBox, VmWare Workstation et Stratus

Les hyperviseurs de type 1 (natifs) accèdent directement au matériel et ne nécessitent pas la prise en charge d'un système d'exploitation sous-jacent. Ils agissent comme le système d'exploitation et contrôlent les appels de ressources.

Les hyperviseurs de type 2 s'exécutent sur un système d'exploitation traditionnel en tant qu'application. Ils fournissent des capacités de virtualisation en coordonnant les appels de ressources via le système d'exploitation hôte.

Voici comment les deux solutions de virtualisation sont classées :

  • VMware ESXi est un hyperviseur de type 1 qui doit être installé sur bare metal.
  • VMware Player, Workstation et Fusion sont des hyperviseurs de type 2 qui doivent être installés sur un périphérique physique avec un système d'exploitation hôte.
  • VirtualBox est également un hyperviseur de type 2 qui doit être installé sur un système d'exploitation hôte.

Forme/lieu de sauvegarde de la machine virtuelle créée

Si vous créez un snapshot de mémoire, vous capturez également l'état de la mémoire de la machine virtuelle. Ces états sont sauvegardés dans des fichiers situés au même emplacement que les fichiers de base de la machine virtuelle. OVA est un format de fichier qui combine les images de disque de la machine virtuelle, la configuration matérielle virtuelle et les métadonnées dans un seul fichier d'archive. L'ordinateur virtuel stocké s'affiche dans le volet Objets de bibliothèque physique.

Explication de tous les choix techniques possibles

On va voir les différentes configurations que nous avons fait pour notre machine virtuelle et expliquer ces différents choix.

  • Général : le système que nous avons choisi est Debian car il fait partie de la suite Linux et que c’est un système assez facile d’usage et de compréhension pour une première utilisation d'une machine virtuelle.
  • Systèmes : Pour la configuration du système, nous avons choisi d'alloués 4096 Mo de mémoire vive et 4 cœurs aux processeurs car ces capacités sélectionnés feront complètement l’affaire pour les tâches que nous allons effectuer.
  • Stockage : Nous avons choisi 20 Go de stockage alloués qui était la mesure de base proposés par VirtualBox et qui sera parfaitement suffisante pour nous et notre utilisation de notre machine.
